## 贡献度量化方法

贡献值综合以下四个维度：

| 维度 | 权重 | 说明 |
|------|------|------|
| 代码贡献（Git commits） | 40% | 有效代码提交行数（去除注释/空行/自动生成代码） |
| Issue 完成质量 | 30% | 负责 Issue 数量 × 难度系数（P0=3分，P1=2分，P2=1分） |
| 设计与架构贡献 | 20% | 技术选型、架构设计、核心难题攻关 |
| 协作与文档贡献 | 10% | Code Review、文档编写、答辩 PPT 制作 |

## 贡献值核算说明

- 贡献值认定以 **Git 提交历史** 为客观依据
- 代码行数通过 `git log --author="X" --numstat` 统计（排除自动生成文件）
- Issue 难度系数由组长与指导老师共同确认
- 所有成员对本贡献分配方案无异议（签字确认）
